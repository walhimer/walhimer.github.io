# Machine Aesthetic Artwork Statement

## Working Description

`Machine Aesthetic` is a single artwork distributed across four code-based works:

- `machine-aesthetic.html` (framework and artistic logic)
- `bloom-four-walls-cathedral.html` (spatial atmospheric system)
- `technical-drawing-with-elevation-2.html` (procedural drafting/language engine)
- `walk_moon_audio_standalone.html` (lifetime arc with persistent celestial continuity)

## Combined Work

The project combines these systems into one coherent artwork that lives in three contexts:

1. **Loop digital context** (`looponland.io` / Loop environment)  
   The work runs as a live generative code object.
2. **LED object context**  
   The same system is embodied as an LED-panel-based physical artifact.
3. **Spatial room context**  
   The installation becomes a room-scale organism where light, sound, and behavior act as one responsive system.

## Core Idea

The code is primary.  
Screenshots, recordings, and objects are outputs or embodiments of the same generative logic.

The four files in this folder should be read as modules of one artwork system, not as separate finished pieces.

## Implementation Rule

All four files should use the same upstream foundations:

- Canonical genome and seeded logic from [walhimer-studio/EmergentDNA](https://github.com/walhimer-studio/EmergentDNA)
- Shared room/network parameter model from [walhimer-studio/walhimer.github.io/studio/light-art-osc](https://github.com/walhimer-studio/walhimer.github.io/tree/main/studio/light-art-osc)
- A unified slider-driven control layer (one state model, one parameter vocabulary, consistent mapping across digital, LED, and room contexts)

Repository reference: [walhimer-studio/walhimer.github.io](https://github.com/walhimer-studio/walhimer.github.io)
