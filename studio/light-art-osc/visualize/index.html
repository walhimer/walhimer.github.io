Light Art OSC — visualize

Tabbed view of the stack: addresses, data flow, and how to run the bridge. UDP OSC cannot be read directly in a browser; this page is the map. Live packets still require a terminal self-test or a future WebSocket bridge.

What you are looking at

Light Art OSC is the contract between Light Art 023 (browser lab: Three.js + Tone.js) and the room: OSC as UDP glue, then Python (or other services) for DMX / Art-Net / sACN. EmergentDNA holds replayable seeds; the browser holds the phenotype.

Why this is not “live OSC” yet. Safari/Chrome cannot bind a UDP socket to sniff OSC. To see packets today, run osc_self_test in Python or Node (Bridge tab). A small local WebSocket relay could mirror UDP into this page later.

Bridge commands

Run from studio/light-art-osc (clone root). Self-test sends 13 messages and expects 13 back.

Python (venv)

cd studio/light-art-osc
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 bridge/osc_self_test.py

Node (osc.js)

cd studio/light-art-osc/bridge
npm install
cd ..
node bridge/osc_self_test.cjs

Two terminals (debug)

python3 bridge/osc_listen.py

Second terminal:

python3 bridge/osc_send_test.py
If you see bind EADDRINUSE, another process is using port 9000. Stop other listeners or use --port 9002 on both sides.

Serve this HTML

cd studio/light-art-osc
python3 -m http.server 8890

Open http://localhost:8890/visualize/
